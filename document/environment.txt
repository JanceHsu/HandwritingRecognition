Environment snapshot for HandwritingRecognition

Date: 2026-05-17

Python runtime:
- Version: 3.12.x (recommended)

Python packages (examples):
- torch: 2.12+ (use CUDA-enabled wheel when training on GPU)
- torchvision: compatible with selected torch version
- opencv-python: 4.x (only used by some scripts; runtime UI uses Qt QImage path)

C++ toolchain (recommended):
- Qt: 6.11.1 (msvc2022_64)
- MSVC: Visual Studio 2022 (Build Tools or full)
- CMake: 3.20+ (or the VS2022 bundled CMake)
- LibTorch: matching the PyTorch version and desired backend (CPU-only or CUDA-enabled)

CUDA / LibTorch notes:
- For GPU training and CUDA inference, use a LibTorch CUDA build that matches your CUDA Toolkit (tested with CUDA 12.8 and LibTorch 2.5.x CUDA builds).
- Ensure `LIBTORCH_DIR` points to the chosen LibTorch; package and `run_handwriting_recog.bat` will copy the matching DLLs into `dist`.

Runtime configuration:
- Default inference device: CPU (toggle CUDA via UI checkbox)
- LibTorch path override: `LIBTORCH_DIR=<path-to-libtorch>`

Paths used for local development in this workspace:
- Qt: D:/Develop/Qt/6.11.1/msvc2022_64
- LibTorch: D:/Develop/libtorch (set `LIBTORCH_DIR` to switch to CUDA build)

Scripts:
